CBR papers and notes

Paper 1: Gentle intro to Case-based reasoning

Primary knowledge source is a memory of stored cases recording specific prior episodes.
Solutions are generated by retrieving the most relevant cases from memory and adapting them to fit new situations.

Cases

Cases may contain prior solutions, or traces of how those solutions were derived.

Retrieval

This includes top-down retrieval: a problem description was formed and used to select a relevant case, and bottom-up: guide retrieval, favoring features that are useful to discriminate between the cases in memory.

Adaption

Correct case adaptation requires that those rules capture both a theory of case adaptation, and the needed aspects of the domain theory to carry out changes. However, there often lacks such a theory.

Similarity assessment

One issue is how to determine the right features to compare.
Similarity assessment can be used to determine which cases are most usefully similar, given the desired results of the CBR process, or aim at facilitating solution generation.

Case evaluation

Evaluation is problematic because evaluating candidate solutions may require considerable domain knowledge and reasoning effort.

Storage

storage of cases includes consideration of efficiency and capacity.

Adaption

Central questions for adaptation are which aspects of a situation to adapt, which changes are reasonable for adapting them, and how to control the adaptation process.

Paper 2: How CBR explains neural networks: A theoretical analysis of XAI Using Post-Hoc Explanation by example from a survey of ANN-CBR Twin-Systems

Twin this ANN with a CBR system using the same dataset, extract the ANN's feature-weights and apply them in the CBR-system to retrieve neighboring cases to the query-case, then we can use the latter to explain the predictions of the former.

Future work:
Deployment of Cases can explore many ways for explanation. More needs to be done on the ideas in the context of ANNs.

Paper 4:

Improving retrieval performance can be done through more effective approaches to similarity assessment.

Cases that are most similar to the target problem may not be the ones with most compelling explanation. Cases that lie between the target problem and the decision boundary can often be more useful for explanation.

When each solution (class) appears frequently in the case base, the most similar retrieved case, if sufficiently similar, is likely to contain a proper solution. But if the retrieved case have significant differences from the new problem, then the solution need to be adapted.
